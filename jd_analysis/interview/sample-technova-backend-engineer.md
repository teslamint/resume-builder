## 2. 라이브 코딩 테스트 대비 (핵심)

1차 면접에서 라이브 코딩 테스트가 진행될 수 있습니다.

### 예상 유형

| 유형 | 가능성 | 맥락 |
|------|--------|----------|
| 알고리즘 | 높음 | 배열/문자열 처리, 해시맵 활용 |
| API 구현 | 높음 | REST 엔드포인트 작성, 에러 처리 |
| SQL 쿼리 | 중간 | JOIN, 집계, 서브쿼리 |
| 리팩토링 | 중간 | 레거시 코드 개선, 테스트 작성 |

### 라이브 코딩 예상 문제 (핵심 3개)

| 유형 | 문제 | 핵심 포인트 |
|------|------|-------------|
| **해시맵 활용** | "중복 요소 찾기, 빈도수 계산" | dict/Counter 활용, O(n) 시간복잡도 |
| **API 구현** | "CRUD 엔드포인트 작성" | 입력 검증, 에러 응답, 상태 코드 |
| **비동기 처리** | "여러 API 동시 호출 후 결과 병합" | asyncio.gather, 예외 처리 |

### 코딩 시 주의사항

**Do's:**
- 문제 이해 확인 후 시작 ("입력은 ~이고, 출력은 ~인 거죠?")
- 엣지 케이스 먼저 언급 (빈 배열, null, 음수 등)
- 생각을 말하면서 코딩 (think aloud)
- 시간/공간 복잡도 설명

**Don'ts:**
- 바로 코드 작성 시작
- 침묵 속에 혼자 생각
- 완벽한 답 찾으려고 오래 고민

### 연습 자료

- [ ] LeetCode Easy/Medium 문제 5개 (Python)
- [ ] FastAPI CRUD 구현 연습
- [ ] SQL JOIN/집계 문제 연습
