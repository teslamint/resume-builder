# JD 스크리닝 분석 스킬

채용공고를 개인 스크리닝 기준에 따라 분석하고 지원 여부를 판단합니다.

## 사전 요구사항

1. 스크리닝 규칙 파일: `job_postings/jd-screening-rules.md`
2. 분석 대상: 채용공고 파일 또는 URL

## 스크리닝 워크플로우

### 1. 입력 확인

- 채용공고 파일 경로 (예: `job_postings/active/*.md`)
- 채용공고 URL (Wanted, Remember 등)
- URL인 경우: `extract-job-posting` 스킬로 먼저 추출

### 2. 스크리닝 규칙 로드

```
Read: job_postings/jd-screening-rules.md
```

### 3. 기업 정보 로드 (있으면)

```
company_info/{company}.md 파일 확인
- 회사 개요, 재무 정보, 복지 등 참고
```

### 4. 6가지 기준 분석

| 기준 | 판정 | 근거 |
|------|------|------|
| 1. 회사 안정성 | ⭕/❌/△ | 중견·대기업/금융·규제 = ⭕, 스타트업 조건 미충족 = ❌ |
| 2. 연봉 구조 | ⭕/❌/△ | 하향 압박 없음 = ⭕, 스톡옵션 대체/포괄임금 = ❌ |
| 3. 리드 전가 여부 | ⭕/❌ | IC 역할 = ⭕, 리드 전가형 = ❌ |
| 4. 운영·정합성 중심 | ⭕/❌ | 운영/안정성/정합성 중심 = ⭕, 성장 드라이브 = ❌ |
| 5. 성장 중심 여부 | ⭕/❌ | 사고 방지 중심 = ⭕, 트래픽 스케일링 = ❌ |
| 6. 도메인 적합성 | ⭕/△/❌ | 강점 활용 가능 = ⭕, 대용량 트래픽 필수 = ❌ |

### 5. 결과 파일 저장

```
jd_analysis/screening/{id}-{company}-{position}.md
```

### 6. 자동 분류 (파일 이동)

스크리닝 완료 후 판정 결과에 따라 JD 파일 자동 분류:

```bash
python3 templates/jd_pipeline.py --classify <jd_file_folder>
```

| 판정 | 대상 폴더 |
|------|----------|
| 지원 추천 | `job_postings/conditional/high/` |
| 지원 보류 | `job_postings/conditional/hold/` |
| 지원 비추천 | `job_postings/pass/` |

### 3. 핵심 문제 분석
{성장 vs 사고 방지 중심 판별}

## 판정 기준 요약

### 즉시 패스 (❌)

- "대규모 트래픽", "수백만 사용자" 핵심 미션
- "X배 성장", "스케일 업" 핵심 목표
- "주도적으로", "오너십", "회색 영역"
- 직함 없이 "리드 경험 우대"
- 코딩테스트/사전 과제 명시
- 평균 연봉 명백히 낮음
- 스톡옵션으로 현금 대체

### 긍정 신호 (⭕)

- 운영, 안정성, 신뢰성
- 정합성, 장애 대응
- 규제/금융/보험/정산
- 기존 시스템 개선/레거시 정리

## 사용 예시

### 파일 기반 분석

```
사용자: "이 채용공고 스크리닝해줘: job_postings/active/254599-wanted.md"
```

### URL 기반 분석

```
사용자: "https://www.wanted.co.kr/wd/254599 스크리닝 분석해줘"

1. extract-job-posting 스킬로 정보 추출
2. jd-screening 규칙으로 분석
3. 결과 파일 저장
```

### 일괄 분석

```
사용자: "job_postings/active/ 폴더의 모든 공고 스크리닝해줘"

1. 폴더 내 모든 .md 파일 목록 조회
2. 각 파일에 대해 스크리닝 분석
3. 결과 요약 테이블 출력
```
